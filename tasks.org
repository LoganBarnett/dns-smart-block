** Production Panic Safety

Rust production code must never panic.  Each task here identifies a call site
that can panic at runtime and proposes a test-first fix.

*** DONE [#A] Remove ~.expect()~ in queue-processor config lookup
CLOSED: [2026-03-28 Fri]

*File:* ~queue-processor/src/main.rs:334~
#+begin_example
.expect("Classifier config should exist for all classification types")
#+end_example

This panics if ~get_classifier_states~ returns a classification type that
has no matching entry in ~config.classifiers~.  This can happen if the
database contains stale classification types from a previous configuration.

*Test first:*
Write a test that calls the processing function with a ~states~ map
containing a classification type not present in ~config.classifiers~.
Assert that the function returns an error rather than panicking.

*Fix:*
Replace ~.expect()~ with ~.ok_or_else(|| ...)~ and propagate the error, or
~continue~ with a warning log if the intent is to skip unknown types.

*** DONE [#A] Remove ~.unwrap()~ on ~last_error~ in classifier retry path
CLOSED: [2026-03-28 Fri]

*File:* ~classifier/src/web_classify.rs:150~
#+begin_example
Err(last_error.unwrap().into())
#+end_example

After the retry loop exhausts all attempts, ~last_error~ is unwrapped.  If
~max_retries~ is 0, the loop body never executes and ~last_error~ remains
~None~, causing a panic.

*Test first:*
Write a test that calls ~fetch_domain_content~ (or the enclosing function)
with ~max_retries = 0~.  Assert that it returns an ~Err~ rather than
panicking.

*Fix:*
Replace the unwrap with a fallback error, e.g.:
#+begin_src rust
Err(last_error
    .map(Into::into)
    .unwrap_or_else(|| ClassifyError::NoRetries))
#+end_src

*** DONE [#C] Document metrics ~lazy_static~ unwraps as acceptable
CLOSED: [2026-03-28 Fri]

*File:* ~blocklist-server/src/metrics.rs~ (16 instances)

The Prometheus ~register_*!~ macros in ~lazy_static!~ blocks use ~.unwrap()~.
These only fail if a metric name is registered twice, which is a programmer
error detected at process startup.  This is the idiomatic Rust-Prometheus
pattern.

*Action:*
Add a module-level doc comment explaining why these unwraps are acceptable
and cannot fail at runtime barring a code-level bug.
